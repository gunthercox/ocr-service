# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.2.x   | :white_check_mark: |
| < 1.2   | :x:                |

We take the security of `ocr-service` seriously. If you discover a security vulnerability, **please do not report it by creating a public GitHub issue.** Instead, please use one of the following methods to report the issue privately:

### 1. Private Vulnerability Reporting (Recommended)

Please report the vulnerability using GitHub's Private Vulnerability Reporting feature:
[Report a Vulnerability](https://github.com/gunthercox/ocr-service/security/advisories/new)

### 2. Email the Maintainer

Alternatively, you can send an email directly to the maintainer at **gunthercx@gmail.com**.

### What to include in your report:

To help us quickly understand and resolve the issue, please include the following information in your report:
* **Description:** A detailed description of the vulnerability.
* **Reproduction Steps:** Step-by-step instructions on how to reproduce the issue (including any sample payloads, specific OCR configurations, or images).
* **Potential Impact:** How this vulnerability could be exploited and its potential risk to users.
* **Suggested Fix:** Any workarounds or ideas you have for mitigating the vulnerability.

### What to expect

* We will strive to acknowledge receipt of your vulnerability report within **[e.g., 48 hours]**.
* We will keep you updated as we investigate the issue and develop a patch.
* Once the vulnerability is resolved, we will coordinate a public disclosure and credit you for the discovery (unless you prefer to remain anonymous).
